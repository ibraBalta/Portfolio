Assistant Airbnb IA

Application hybride d'IA Générative et Prédictive pour l'analyse immobilière

Sommaire du rapport

  • 01. IA Générative : Création de contenu & Tourisme

Ce projet universitaire vise à développer une application complète d'aide à la décision pour les hôtes Airbnb. Baptisée Assistant Airbnb IA, cette plateforme combine deux approches de l'intelligence artificielle : une IA générative pour la rédaction de contenu marketing et touristique, et une IA prédictive pour l'estimation tarifaire des biens.

L'application, développée sous Streamlit, permet d'analyser un logement spécifique, d'enrichir sa description grâce à des données géolocalisées (API Google Places) et d'obtenir une prédiction de prix fiable via un modèle CatBoost Regressor. Ce projet hybride lie les compétences de Data Science (nettoyage, feature engineering, modélisation) avec le développement web et l'intégration d'APIs.

IA Générative : Création de contenu & Tourisme

Pipeline de données : De la géolocalisation au texte

La première brique du projet transforme des données techniques brutes en contenu rédactionnel attractif. L'utilisateur sélectionne un logement et obtient instantanément une description marketing optimisée et un guide touristique local. J'ai conçu un pipeline qui connecte les caractéristiques du logement à l'API Google Places pour récupérer les points d'intérêt (restaurants, musées) et nourrir un modèle de langage.

Voici un extrait du code Python gérant la récupération des lieux touristiques via l'API, étape indispensable pour éviter les "hallucinations" de l'IA en lui fournissant des lieux réels :

def get_nearby_places(lat, lng, radius=1000, type_place="tourist_attraction"):
    endpoint = "https://maps.googleapis.com/maps/api/place/nearbysearch/json"
    params = {
        "location": f"{lat},{lng}",
        "radius": radius,
        "key": GOOGLE_API_KEY,
        "language": "fr"
    }
    response = requests.get(endpoint, params=params)

    # Extraction et nettoyage des données pertinentes
    places = []
    for place in response.json().get('results', [])[:5]:
        places.append({
            "nom": place.get("name"),
            "rating": place.get("rating", "N/A"),
            "adresse": place.get("vicinity")
        })
    return places

IA Prédictive : Estimation des prix (Machine Learning)

Modélisation avec CatBoost Regressor

La seconde composante majeure est l'estimation du prix de la nuitée. J'ai opté pour l'algorithme CatBoost (Categorical Boosting), particulièrement performant sur les données tabulaires contenant de nombreuses variables catégorielles comme le quartier ou le type de chambre.

Le Feature Engineering a été une étape clé : extraction de mots-clés, calcul de l'ancienneté, et transformation logarithmique de la cible (Log-Price) pour stabiliser la variance. L'interface permet ensuite d'ajuster les hyperparamètres en temps réel.

# Configuration dynamique du modèle via Streamlit
learning_rate = st.sidebar.slider("Taux d'apprentissage", 0.01, 0.3, 0.1)
depth = st.sidebar.slider("Profondeur de l'arbre", 4, 10, 6)

model = CatBoostRegressor(
    iterations=500,
    learning_rate=learning_rate,
    depth=depth,
    loss_function='RMSE'
)

# Entraînement et Prédiction
model.fit(X_train, y_train, cat_features=cat_cols)
preds = model.predict(X_test)
⚡ Résultats et Métriques

Grâce à l'optimisation des hyperparamètres via l'interface, le modèle final atteint des performances robustes sur le jeu de test :

  • RMSE (Erreur Quadratique Moyenne) : 0.24 (sur échelle log)
  • R² (Coefficient de détermination) : 0.78
  • Variables clés : Localisation (Quartier), Type de logement, Capacité d'accueil.
Bilan Technique : Ce projet illustre la convergence entre Data Engineering (API, nettoyage), Data Science (modélisation prédictive) et Développement Web. J'ai appris à déployer un modèle de ML dans une application interactive, rendant les prédictions intelligibles pour l'utilisateur, tout en enrichissant l'expérience grâce à la génération de texte automatisée.
